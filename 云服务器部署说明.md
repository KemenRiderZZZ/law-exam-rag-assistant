# 云服务器部署说明

当前项目已经按“运行时最小集合”和“本地资料集合”做了拆分准备。

## 当前推荐入口

优先使用：

```bash
docker compose -f docker-compose.cloud.yml up -d --build
```

这套根目录编排不会再直接依赖整仓运行，而是改为只读取：

- `云端部署/runtime/数据库`
- `云端部署/runtime/法考对话前端2.0/dist`

## 运行时目录

真正推荐上传到云服务器的是：

```text
云端部署/runtime
```

这个目录里只保留：

- Python 查询服务
- 检索依赖脚本
- 前端 `dist`
- 数据库初始化 SQL
- 云端部署所需环境变量模板
- Docker / Compose 文件

## 不建议上云的目录

以下内容属于资料生产链或历史内容，不建议作为运行时上传：

- `OCR原稿`
- `整理后文本`
- `切块`
- `保密版文本`
- `保密版切块`
- `法考对话前端`
- `页面`
- `备份`
- `运行日志`
- `工具`
- `脚本`

## 数据准备方式

这次默认推荐：

1. 本地先导出已经可用的 `lawqa`
2. 云端恢复数据库
3. 再启动应用服务

不推荐把 OCR、整理稿、切块文件一起搬上云再重建。

## 后续正式部署

部署前的建议顺序：

1. 先生成并检查 `云端部署/runtime`
2. 填好：
   - `云端部署/runtime/数据库/.env.pg`
   - `云端部署/runtime/数据库/.env.embedding`
3. 导出现成数据库
4. 云端恢复数据库
5. 启动 `docker-compose.cloud.yml`

## 后续仍建议追加

- Nginx 反向代理
- 域名
- HTTPS 证书
- 应用与数据库备份策略
