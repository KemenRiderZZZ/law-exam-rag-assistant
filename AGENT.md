# 法考智学项目 Agent 指南

## 项目定位
- 这是一个法考问答 + 搜索 + 导图生成项目。
- 前端：`法考对话前端2.0`
- 后端：`数据库/search_web_server.py`
- 云端运行包：`云端部署/runtime`

## 工作原则
- 先看现状，再改代码；不要凭经验直接猜。
- 尽量保持桌面端、手机 Chrome、线上现有体验不变，除非用户明确要求改。
- 只改当前任务相关文件，不要顺手重构无关模块。
- 不要删除用户没要求删的资源、数据、历史备份。
- 需要改文件时优先用 `apply_patch`。
- 需要查找时优先用 `rg`。

## 重点关注点
- `/api/search`、`/api/chat`、`/api/mindmap`、`/api/traffic-summary`、`/api/health` 不要随意改坏。
- 导图相关改动要兼容：
  - 桌面端原有体验
  - 手机 Chrome 现有 HTML 导出
  - 弱兼容浏览器的兜底预览
- 线上问题优先查：
  - `D:\桌面\云端法考知识问答项目\云端部署\runtime`
  - 服务器 `/var/log/nginx/access.log`
  - Docker 日志

## 部署约定
- 前端改完先执行：
  - `npm run lint`
  - `npm run build`
- 然后把最新 `dist` 同步到：
  - `云端部署/runtime/法考对话前端2.0/dist`
- 线上发布按现有 Docker 流程走，不要保留旧 bundle。

## 运维习惯
- 如果用户说“又卡了 / 又打不开 / 又不对”，先检查：
  - 是否还是旧 bundle
  - 是否缓存没清
  - 是否是浏览器兼容问题
  - 是否是生产日志里根本没收到请求
- 用户常会关心：
  - 最近真实搜索记录
  - 今日访问量
  - 导图是否能预览
  - 线上是否还是旧版本

## 回复风格
- 直接给结论，再给必要证据。
- 少说空话，多说可执行结果。
- 如果有风险或不确定，先说明再动手。

## 额外约定
- 不要把管理员功能暴露给普通用户。
- 统计类信息优先做成后台汇总，不要做成所有人都能看到的管理页。
- 如果需要新增长期规则，优先写进这个文件。
